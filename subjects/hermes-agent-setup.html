# How to Setup WordPress with Hermes

Blueprint for connecting Hermes Agent to a WordPress blog (e.g., Hostinger-hosted) to research topics and write/publish posts autonomously.

---

## Phase 1 — Enable WordPress REST API

On the WordPress side:

1. **Confirm your site URL** — you'll need it for all API calls
2. **Generate an Application Password**:
   - Go to WP Admin → Users → Profile → Application Passwords
   - Enter a name (e.g., "Hermes Agent") and click **Add New**
   - Save the generated password — it only shows once
   - *No plugin needed — Application Passwords are a WordPress core feature and work on Hostinger out of the box*
3. **Verify the REST API** works by visiting: `https://yourdomain.com/wp-json/wp/v2/posts`
4. **Check Permalinks** — ensure they are *not* set to "Plain"

# Memory OS — A 6-Layer Open-Source Memory Stack for Hermes Agent

**Source:** [MarkTechPost — June 1, 2026](https://www.marktechpost.com/2026/06/01/meet-memory-os-a-6-layer-open-source-memory-stack-built-on-top-of-hermes-agent/)

## Overview

Hermes Agent already remembers across sessions with curated memory files and full-text session search. But a new community project — **Memory OS** — argues built-in memory is too shallow for serious work. It stacks **six memory layers** onto Hermes, adding a vector database, structured facts, and an auto-curated knowledge wiki.

Memory OS is not a Hermes plugin you toggle on.
