## Overview

Hermes Agent already remembers across sessions with curated memory files and full-text session search. But a new community project — **Memory OS** — argues built-in memory is too shallow for serious work. It stacks **six memory layers** onto Hermes, adding a vector database, structured facts, and an auto-curated knowledge wiki.

Memory OS is not a Hermes plugin you toggle on.

Memory OS Stack for Hermes Agent

Researched Memory OS: new MIT-licensed open-source 6-layer memory stack built on Hermes Agent. Layers: Workspace (MEMORY.md), Sessions (state.db + FTS5), Structured Facts (SQLite + trust scoring), Fabric (forked Icarus, 16 tools), Qdrant vector DB (4096d), LLM Wiki (auto-curated). Features 4-level fallback cascade, social-closer filter, weekly decay scanner, local-first Docker stack.
