# Image & Video Generation Models for Local AI PCs

Comprehensive guide to running image and video generation models locally on **AMD Strix Halo systems** (Ryzen AI Max+ 395, Radeon 8060S 40 CU, 256 GB/s unified memory).

> **Key difference from LLMs:** These models need GPU compute (not just memory bandwidth). The 8060S iGPU is roughly **RTX 4060 laptop** class in raw compute — capable but not a datacenter GPU. Generation times will be longer than cloud services, but you own the pipeline and pay $0 per image.

**Runner: ComfyUI** is the standard for local image/video gen. Supports all these models via custom nodes. **Diffusers** (HuggingFace) for Python pipelines.

# Local LLM Models for AI Mini PCs — Memory Tier Guide

Comprehensive guide to running LLMs locally on AMD Strix Halo systems (Ryzen AI Max+ 395) with unified LPDDR5X-8000 memory at **256 GB/s bandwidth**. Covers the four key memory tiers: **32GB, 64GB, 96GB, and 128GB**.

> **Scope:** GGUF quantization via llama.cpp / Ollama / LM Studio. Vulkan/ROCm backend. All sizes approximate — actual GGUF files vary by tokenizer, vocabulary, and architecture.

---

## ⚡ Quick Reference: Quantization Sizes

| Quantization | Size Formula | Quality | Best For |
|---|---|---|---|
| **Q4_K_M** | ~0.55 GB per 1B params | Very good (95%+ of FP16) | Daily driver, best balance |

| **Q8_0** | ~1.0 GB per 1B params | Near-lossless | Critical tasks, math |

| **Q3_K_M** | ~0.42 GB per 1B params | Acceptable | Last resort for giants |
| **IQ2_XXS** | ~0.28 GB per 1B params | Degraded | Extreme size reduction |
