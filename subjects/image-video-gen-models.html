Image & Video Generation Models for Local AI PCs


Comprehensive guide to running image and video generation models locally on <strong>AMD Strix Halo systems</strong> (Ryzen AI Max+ 395, Radeon 8060S 40 CU, 256 GB/s unified memory).
**Key difference from LLMs:** These models need GPU compute (not just memory bandwidth). The 8060S iGPU is roughly **RTX 4060 laptop** class in raw compute — capable but not a datacenter GPU. Generation times will be longer than cloud services, but you own the pipeline and pay $0 per image.

<strong>Runner: ComfyUI</strong> is the standard for local image/video gen. Supports all these models via custom nodes. <strong>Diffusers</strong> (HuggingFace) for Python pipelines.

    Overview


    Hermes Agent already remembers across sessions with curated memory files and full-text session search. But a new community project — <strong>Memory OS</strong> — argues built-in memory is too shallow for serious work. It stacks <strong>six memory layers</strong> onto Hermes, adding a vector database, structured facts, and an auto-curated knowledge wiki.
    Memory OS is not a Hermes plugin you toggle on. It's a <strong>layered system that sits beside Hermes Agent's own memory</strong>, keeping the built-in layers and adding four more above them. The full stack runs locally using Docker, Qdrant, Redis, and Python 3.11+.

    Local LLM Models for AI Mini PCs — Memory Tier Guide


    Comprehensive guide to running LLMs locally on AMD Strix Halo systems (Ryzen AI Max+ 395) with unified LPDDR5X-8000 memory at <strong>256 GB/s bandwidth</strong>. Covers the four key memory tiers: <strong>32GB, 64GB, 96GB, and 128GB</strong>.
    **Scope:** GGUF quantization via llama.cpp / Ollama / LM Studio. Vulkan/ROCm backend. All sizes approximate — actual GGUF files vary by tokenizer, vocabulary, and architecture.

    • ⚡ Quick Reference: Quantization Sizes


    | Quantization | Size Formula | Quality | Best For | |---|---|---|---| | **Q4_K_M** | ~0.55 GB per 1B params | Very good (95%+ of FP16) | Daily driver, best balance | | **Q5_K_M** | ~0.65 GB per 1B params | Excellent | When you have headroom | | **Q8_0** | ~1.0 GB per 1B params | Near-lossless | Critical tasks, math | | **IQ4_XS** | ~0.45 GB per 1B params | Good | Squeezing big models in | | **Q3_K_M** | ~0.42 GB per 1B params | Acceptable | Last resort for giants | | **IQ2_XXS** | ~0.28 GB per 1B params | Degraded | Extreme size reduction |
